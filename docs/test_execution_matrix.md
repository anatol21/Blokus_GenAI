# Test Execution Matrix

This document is the normalized execution view for the current Classic baseline.

- Source of truth for requirement origin and broad traceability: `docs/requirements_source_matrix.md`
- Source of truth for concrete package ownership: `OWNERSHIP.md`
- Scope of this execution matrix: Classic-only executable coverage plus explicit Duo deferral
- Duo status in this matrix: design-only and intentionally not implemented in this change

Scenario rows are primary by design, so the row `Scenario ID` is the primary scenario identifier. Reuse is tracked through `Referenced Scenario IDs` and `Cross-Coverage Notes` rather than by restating the same scenario text in many places.

## Quantified Threshold Refinement Matrix

| Requirement ID | Current wording | Proposed measurable threshold | Evidence / Execution Path | Status |
| --- | --- | --- | --- | --- |
| `R-NF-15` | AI move response should be fast enough to preserve flow | `choose_simple_move()` returns within `<= 250 ms` on the canonical Classic baseline fixture; CLI `suggest` returns within `<= 1 s` locally | Add a small benchmark or timing assertion against a fixed fixture once timing policy is approved | Proposed, executable once timing guard is adopted |
| `R-NF-18` | New user can play meaningfully within minutes | Guided path reaches the first valid move in `<= 6 user actions` and `<= 120 s` from entry to gameplay | Manual onboarding test or future GUI automation | Proposed, needs product confirmation |
| `R-NF-06` | Interface is simple and immediately understandable | New game, legal first move, save/resume, and help are reachable within `<= 3 primary UI actions` from the main play surface, with no hidden mandatory navigation | Manual UI review checklist or future GUI automation | Proposed, needs UX confirmation |
| `R-NF-10` | UI should be reassuring for low-confidence users | First-use controls have tap targets `>= 44x44 px`, tutorial copy stays within short actionable prompts, and common mistakes never lead to an unrecoverable dead end | Design review plus manual UI inspection | Proposed, needs UX confirmation |
| `R-NF-16` | No lag in navigation/board interaction | Local placement feedback appears within `<= 100 ms` for standard actions on baseline hardware | Future GUI instrumentation or manual profiling | Proposed, needs instrumentation |
